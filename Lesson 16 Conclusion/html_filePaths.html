HTML - File Paths

A file path defines the location of a file within a website’s folder structure. File paths are used in HTML to link to external resources such as images, style sheets, and JavaScript files.

1. File Path Examples

Path Description
<img src="picture.jpg"> The file picture.jpg is located in the same folder as the current page.
<img src="images/picture.jpg"> The file is in the images subfolder of the current folder.
<img src="/images/picture.jpg"> The file is located in the images folder at the root of the website.
<img src="../picture.jpg"> The file is located one folder up from the current directory.

2. Understanding File Paths

File paths define where a file is located relative to the current HTML document or as an absolute web address.

They are used to connect to external resources such as:

  • Web pages (<a href>)
  • Images (<img src>)
  • Style sheets (<link href>)
  • JavaScript files (<script src>)

3. Absolute File Paths

An absolute file path provides the full URL of the file — including the domain name. This type of path is used when the file is hosted on another server or external website.

Example:

<img src="https://www.w3schools.com/images/picture.jpg" alt="Mountain">

Here, the image is loaded directly from the full web address.

4. Relative File Paths

A relative file path locates a file relative to the current HTML file’s location. This is the preferred approach for local and consistent site management.

Example 1 – Root Folder

<img src="/images/picture.jpg" alt="Mountain">

The file is in the images folder at the root of the site.

Example 2 – Current Folder

<img src="images/picture.jpg" alt="Mountain">

The file is in an images folder within the same directory as the current page.

Example 3 – One Folder Up

<img src="../images/picture.jpg" alt="Mountain">

The file is located in the images folder one level above the current directory.

5. Best Practices

  • Use relative paths whenever possible. This ensures flexibility when moving your project between environments (e.g., from localhost to a production server).
  • Avoid using absolute URLs unless the resource is external or hosted on a different domain.
  • Keep your folder structure organized (e.g., images/, css/, js/).
  • Test links locally before deployment to ensure file references are correct.

6. Chapter Summary

Path Type Description
Absolute Path Specifies the full URL, including the protocol and domain.
Relative Path Specifies the file location relative to the current document’s folder.
Root-relative Path Starts from the website’s root directory (begins with /).
Parent-relative Path Moves up one level using ../.
