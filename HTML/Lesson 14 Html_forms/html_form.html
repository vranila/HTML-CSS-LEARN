HTML Forms

An HTML form is used to collect user input. The user input is most often sent to a server for processing.

Example:





1. The <form> Element

The HTML <form> element is used to create an input form for users.

Syntax:

<form>
  ... form elements ...
</form>

The <form> tag is a container for multiple types of input elements such as text fields, checkboxes, radio buttons, and submit buttons.

2. The <input> Element

The HTML <input> tag is the most commonly used form element. Its behavior depends on the type attribute.

Common Input Types

TypeDescription
<input type="text">Displays a single-line text input field
<input type="radio">Displays a radio button (one of many choices)
<input type="checkbox">Displays a checkbox (zero or more choices)
<input type="submit">Displays a submit button
<input type="button">Displays a clickable button

3. Text Fields

<input type="text"> defines a single-line input field for text.

Example:

<form>
  <label for="fname">First name:</label><br>
  <input type="text" id="fname" name="fname"><br>
  <label for="lname">Last name:</label><br>
  <input type="text" id="lname" name="lname">
</form>

Note: The form itself is not visible. The default width of an input field is 20 characters.

4. The <label> Element

The <label> element defines a label for form controls.

  • Improves accessibility for screen readers.
  • Makes small clickable elements like checkboxes easier to select.
  • The for attribute in <label> must match the id of its corresponding input.

Example:

<label for="email">Email:</label>
<input type="text" id="email" name="email">

5. Radio Buttons

<input type="radio"> creates radio buttons for selecting one option from a group.

Example:

<p>Choose your favorite web language:</p>
<form>
  <input type="radio" id="html" name="fav_language" value="HTML">
  <label for="html">HTML</label><br>
  <input type="radio" id="css" name="fav_language" value="CSS">
  <label for="css">CSS</label><br>
  <input type="radio" id="javascript" name="fav_language" value="JavaScript">
  <label for="javascript">JavaScript</label>
</form>

Note: All radio buttons with the same name attribute belong to the same group.

6. Checkboxes

<input type="checkbox"> defines a checkbox for selecting one or more options.

Example:

<form>
  <input type="checkbox" id="vehicle1" name="vehicle1" value="Bike">
  <label for="vehicle1">I have a bike</label><br>
  <input type="checkbox" id="vehicle2" name="vehicle2" value="Car">
  <label for="vehicle2">I have a car</label><br>
  <input type="checkbox" id="vehicle3" name="vehicle3" value="Boat">
  <label for="vehicle3">I have a boat</label>
</form>

7. The Submit Button

<input type="submit"> defines a button that submits the form data to a server.

Example:

<form action="/action_page.php">
  <label for="fname">First name:</label><br>
  <input type="text" id="fname" name="fname" value="John"><br>
  <label for="lname">Last name:</label><br>
  <input type="text" id="lname" name="lname" value="Doe"><br><br>
  <input type="submit" value="Submit">
</form>

Note: The form data is sent to the URL specified in the action attribute for processing.

8. The name Attribute

Every input field should have a name attribute — this is the key used to send its value to the server.

Example (missing name attribute):

<form action="/action_page.php">
  <label for="fname">First name:</label><br>
  <input type="text" id="fname" value="John"><br><br>
  <input type="submit" value="Submit">
</form>

Note: Without the name attribute, the field’s value will not be sent when submitting the form.

9. Chapter Summary

  • <form> defines the form area.
  • <input> creates interactive input fields.
  • <label> improves accessibility and usability.
  • <input type="radio"> allows single-choice selection.
  • <input type="checkbox"> allows multiple selections.
  • <input type="submit"> sends form data to the server.
  • name attribute is required for data submission.
