HTML Form Attributes

HTML form attributes define how form data is sent, processed, and displayed after submission. They control behavior such as the form’s action, method, target, and validation.

1. The action Attribute

The action attribute defines the destination where form data is sent for processing when the form is submitted.

Usually, this points to a server-side script (e.g., a PHP or Python file).

Example:

On submit, send form data to "action_page.php":

<form action="/action_page.php">
  <label for="fname">First name:</label><br>
  <input type="text" id="fname" name="fname" value="John"><br>
  <label for="lname">Last name:</label><br>
  <input type="text" id="lname" name="lname" value="Doe"><br><br>
  <input type="submit" value="Submit">
</form>

Tip: If the action attribute is omitted, the form data is submitted to the same page that contains the form.

2. The target Attribute

The target attribute specifies where to display the response received after submitting the form.

Available Values:

ValueDescription
_blankOpens the response in a new window or tab.
_selfOpens the response in the current window (default).
_parentOpens the response in the parent frame.
_topOpens the response in the full body of the window.
framenameOpens the response in a specified iframe.

Example:

Here, the submitted result will open in a new browser tab:

<form action="/action_page.php" target="_blank">
  ...
</form>

3. The method Attribute

The method attribute specifies the HTTP method used to send form data to the server.

There are two main methods: GET and POST.

Example (GET method):

<form action="/action_page.php" method="get">
  ...
</form>

Example (POST method):

<form action="/action_page.php" method="post">
  ...
</form>

Notes on GET:

  • Appends form data to the URL in name/value pairs.
  • Should not be used for sensitive data (visible in URL).
  • URL length is limited (approximately 2048 characters).
  • Useful for bookmarking form submissions or simple search queries.

Notes on POST:

  • Appends form data in the body of the HTTP request (not visible in URL).
  • Can send large amounts of data without size limits.
  • More secure for sensitive or personal information.
  • Cannot be bookmarked.

Tip: Always use POST when dealing with passwords or confidential information.

4. The autocomplete Attribute

The autocomplete attribute specifies whether the browser should automatically complete form fields based on previously entered values.

Syntax:

<form action="/action_page.php" autocomplete="on">
  ...
</form>
ValueDescription
onEnables autocomplete (default).
offDisables autocomplete.

5. The novalidate Attribute

The novalidate attribute is a boolean attribute that prevents the browser from validating the form before submission.

Example:

<form action="/action_page.php" novalidate>
  ...
</form>

Note: When novalidate is used, the browser skips HTML5 form validation rules like required, pattern, or type.

6. Chapter Summary

  • action → Specifies where to send the form data.
  • target → Defines where the response appears.
  • method → Defines the HTTP method (GET or POST).
  • autocomplete → Enables or disables browser autocomplete.
  • novalidate → Disables built-in HTML form validation.
