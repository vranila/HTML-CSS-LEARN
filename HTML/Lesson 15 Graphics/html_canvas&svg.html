HTML Canvas Graphics

The HTML <canvas> element is used to draw graphics on a web page using JavaScript. It acts as a blank drawing surface that can display shapes, text, gradients, and images dynamically.

What is HTML Canvas?

The <canvas> element is only a container — JavaScript actually performs the drawing operations.

  • Canvas supports paths, rectangles, circles, text, and images.
  • All major browsers support the <canvas> element.

Canvas Example

Explanation: moveTo() sets the starting point, lineTo() sets the end point, and stroke() draws the line on the canvas.

2. Draw a Circle

3. Draw Filled Text

Explanation: fillText() renders filled text with the current fillStyle color.

4. Draw Stroke Text

Explanation: strokeText() draws only the border of the text, unlike fillText().

5. Draw Linear Gradient

Explanation: createLinearGradient() creates a smooth transition between two colors. addColorStop() defines where each color appears.

6. Draw Radial (Circular) Gradient

Explanation: createRadialGradient() defines a circular color transition from an inner to outer radius.

7. Draw Image

Explanation: drawImage(image, x, y) places an image at a specified position on the canvas. You can use hidden <img> elements or load images dynamically.

Summary

  • <canvas> creates a drawing area.
  • All drawings use the CanvasRenderingContext2D API.
  • stroke() draws outlines; fill() fills areas.
  • Supports gradients, images, shapes, and text rendering.
  • JavaScript is required for all canvas operations.
HTML SVG Graphics

HTML SVG Graphics

SVG (Scalable Vector Graphics) defines vector-based graphics using XML. These graphics can be directly embedded inside an HTML page and are resolution-independent.

What is SVG?

  • SVG stands for Scalable Vector Graphics.
  • It is used to create vector-based graphics for the web.
  • SVG defines graphics in XML format, making them both human- and machine-readable.
  • Every element and attribute can be animated using CSS or JavaScript.
  • SVG is a W3C recommendation and integrates with CSS, DOM, and JS.
  • Supported by all major browsers.

The <svg> Element

The <svg> element is a container for SVG shapes such as circles, rectangles, paths, and text. You can control colors, borders, and positions with attributes.

1. SVG Circle

Explanation: <circle> draws a circle. Attributes:

  • cx – X-coordinate of center
  • cy – Y-coordinate of center
  • r – radius
  • stroke – border color
  • stroke-width – border thickness
  • fill – inner color

2. SVG Rectangle

Explanation: <rect> draws a rectangle. Attributes: x, y, width, height, stroke, fill.

3. SVG Ellipse

Explanation: <ellipse> is like a circle stretched horizontally or vertically.

4. SVG Line

Explanation: <line> creates a straight line using start (x1, y1) and end (x2, y2) coordinates.

5. SVG Polygon

Explanation: <polygon> connects multiple points to form closed shapes like triangles, pentagons, etc.

6. SVG Polyline

Explanation: <polyline> is similar to <polygon> but doesn’t close the path.

7. SVG Text

Explanation: <text> places text inside SVG and supports all font styling.

8. SVG with Gradient Fill

Explanation: Gradients in SVG are defined inside <defs> and applied using fill="url(#id)".

9. SVG Animation Example

Explanation: <animate> allows SVG elements to move or change attributes over time.

Summary

  • SVG uses XML syntax to describe 2D graphics.
  • SVG is scalable, lightweight, and supports animation.
  • Common shapes: circle, rect, ellipse, line, polygon, polyline, and text.
  • Fully compatible with CSS and JavaScript.
