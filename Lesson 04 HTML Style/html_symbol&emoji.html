HTML Symbols and Emojis

HTML provides a way to display special symbols, mathematical notations, and even emojis that are not available on a regular keyboard. These are added using HTML entities or Unicode character references.

1. HTML Symbol Entities

Symbols that are not present on your keyboard can be added to an HTML page using entities.

Example:

<p>I will display &euro;</p>
<p>I will display &#8364;</p>
<p>I will display &#x20AC;</p>

Output:

I will display €
I will display €
I will display €

6. The HTML charset Attribute

To display an HTML page correctly, the browser must know the character set used on the page.

This is specified using the <meta> tag.

Example:

<meta charset="UTF-8">

If not specified, UTF-8 is the default character set in HTML5.

7. UTF-8 Characters

UTF-8 is the most common character encoding used on the web. It supports all characters, symbols, and emojis.

Many UTF-8 characters cannot be typed directly on a keyboard, but can be displayed using entity numbers.

Example:

<!DOCTYPE html>
<html>
<meta charset="UTF-8">
<body>

<p>I will display A B C</p>
<p>I will display &#65; &#66; &#67;</p>

</body>
</html>

Output:

I will display A B C
I will display A B C

Explanation:

  • The <meta charset="UTF-8"> element defines the character encoding.
  • The entity numbers &#65;, &#66;, &#67; represent A, B, and C respectively.
  • All entity codes must start with &# and end with a semicolon (;).

8. Chapter Summary

  • HTML symbols and emojis use entity names or Unicode values for display.
  • Emojis can be rendered using decimal Unicode references like &#128512;.
  • Use <meta charset="UTF-8"> to ensure proper rendering of characters.
  • UTF-8 supports global languages, symbols, and emojis consistently across browsers.
